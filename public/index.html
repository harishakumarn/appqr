One QR.
Two stores.
No subscription.

Generate a smart QR code that sends iOS users to the App Store and Android users to Google Play. Runs on your machine. Lives on your website. Works forever.

↓ Download see how it works
$ npm install -g appqr
// how it works
01 — ONCE
Run setup
Paste your App Store and Play Store URLs. AppQR generates a tiny redirect.html and your first QR code.
02 — ONCE
Upload one file
Drop redirect.html anywhere on your existing website. You never touch it again.
03 — FOREVER
Run campaigns
Generate new QR codes per campaign with separate Apple and Google tracking params. No new files to upload.
04 — MAGIC
It just works
Phone scans QR → hits your site → detects device → lands on the right store. Every time.
// usage
# First time — generates redirect.html + qr.png
appqr setup \
  --ios "https://apps.apple.com/app/id123" \
  --android "https://play.google.com/store/apps/details?id=com.app" \
  --url "https://myapp.com/go"
 
✓ redirect.html → ./appqr-output/redirect.html
✓ qr.png      → ./appqr-output/qr.png
→ Upload redirect.html to your site at /go
 
# Every campaign after — just a new QR, no upload needed
appqr campaign \
  --url "https://myapp.com/go" \
  --ios-params "ct=instagram_launch&pt=12345&mt=8" \
  --android-params "utm_source=instagram&utm_campaign=launch"
 
✓ qr.png → ./appqr-campaign/qr.png
→ Points to: myapp.com/go?d=eyJhbmRyb2lk...

Agent-friendly by design

AI agents building your marketing site can call AppQR directly as an npm package — no GUI, no clicks, no human needed.

import { setup, campaign } from 'appqr'
 
// agent scaffolding a new marketing site
await setup({ ios, android, hostedUrl, outputPath })
 
// agent generating a campaign QR automatically
await campaign({ hostedUrl, iosUrl, androidUrl, outputPath })
